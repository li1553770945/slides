# 准备篇：打好地基

## 1.1 简历构建与迭代

+ 简历的核心逻辑是：**在最短时间内，让 HR 看到你最匹配的价值。** 

+ 互联网简历侧重内容，信息清晰传达即可，无需过度美化。

+ 推荐简历制作工具：[WonderCV](https://www.wondercv.com/)。

<br>

### 头部信息：清晰对齐
* **基本信息**：姓名、求职意向（开发 or 算法？后端 or 前端？）、联系方式、个人主页或github主页（加分项）。照片可放可不放。
* **教育背景**：学校、专业、绩点 (若有优势，也可写部分课程)、起止时间、所获荣誉（可选），按照时间排序。
> **原则**：固定格式，无需设计感，确保信息易读。

---

### 核心板块

实习经历、核心项目、论文/专利、竞赛奖项。

+ 对于开岗，实习和项目一般必须有，论文、专利、竞赛奖项根据实际情况；算法岗则更侧重论文。
+ 根据个人情况与要找的岗位，按照重要性排序。
+ 不需要罗列所有经历。只选最出彩的，并按重要程度，每个板块只写2-3个项目。
+ 采用 **STAR 原则**，每条经历可以分成 3 个要点，侧重量化和结果。
+ 方法可以不用过于详细，面试时介绍详细解决方案。

```
编写 Go/Envoy 插件完成 Higress 网关 的 MCP 协议适配，将内部微服务以 MCP 暴露，支撑数百个智能体通过 MCP 协议直接调用内部微服务。

使用交叉编译重构插件构建链路并设计 CI 流程，插件单测耗时由约 10 分钟降至 1 分钟，解决由于 DinD 环境父子容器不一致导致 redis 丢包问题。
```

### 补充板块

个人技能、学生工作、社团经历、个人兴趣。

+ 个人技能可写3行左右，表明自己的技术栈，主要目标是**提高岗位匹配度**。
+ 大厂视角：研发岗对学生工作关注度较低。
+ 精简至 1-2 行，除非该经历能体现极强的组织能力或领导力。

```markdown
# 个人技能
容器技术与微服务架构：熟悉 Docker 和 k8s 容器技术及实现原理，曾使用 C++ 实现简易容器运行时；理解微服务架构
AI及Agent：理解 AI 模型和 Agent 基础，能够使用 pytorch 编写和调试神经网络代码，能使用 LangGraph 框架开发 Agent

# 组织经历
负责 NOIP、ICPC 等多个计算机竞赛的技术支持，赛前为选手安装并调试竞赛环境；竞赛中及时解答考生技术性问题，并及时处理赛场设备和网络故障。
```

##  简历总结

<v-clicks>

- **拒绝“流水账”**：面试官不关心你“做了什么”（最多用一个点说明自己的工作内容），更关心你“解决了什么问题、起到了什么作用、达到了什么效果”。
- **量化意识**：将“优化了插件编译速度”改为“插件编译时间从10分钟降低到1分钟”。
- **技术栈对齐**：简历上的每一个字都可能成为考点，不熟悉的的项目、技能千万别随便写。
- **排版技巧**：一页纸原则，开头结尾一般固定，中间核心板块按照重要性排序。

</v-clicks>

# 1.2 项目、实习、八股、刷题

## 项目

+ 准备2~3个项目，不侧重数量，主要侧重质量。
+ 可先根据自己个个人兴趣做，如果没有合适的，可以从平台里面找。

## 实习

+ 目前互联网领域实习尤为重要，可在**完成论文第一个点征得老师同意**后实习。
+ 建议尽快完成第一个点，实习经历需要3个月，而8月就会开始秋招面试。
+ 实习也可以多投一些，一方面积累面试经验，一方面实习offer有可能对于秋招很重要。
+ 实习需要好好找，因为实习转正的难度远小于秋招，而且实习转正拿到的offer对团队知根知底，也更好一点。

## 八股

+ 需要多看看有个知识广度，但只有自己熟悉的东西才需要真的好好记和背，面试的时候往自己熟悉的领域引导面试官。

+ 推荐平台： [小林coding](https://www.xiaolincoding.com/)（官网）、黑马程序员（B站）。

## 算法刷题

+ LeetCode [hot100](https://leetcode.cn/problem-list/2cktkvj/)、[热题 100](https://leetcode.cn/studyplan/top-100-liked/)、[面试150](https://leetcode.cn/studyplan/top-interview-150/)，只要前面交流没啥问题，出题基本不会超出这个范围。
+ 如果题目难度明显很难，表明面试官意愿不太大，但如果真能做出来也会有转机。
+ 一定要刷熟练，如果只知道思路，面试时紧张不一定能调出来。

## 1.3 寻找可投递岗位

互联网大厂：直接官网搜索和投递，最好找认识人内推，可以提前了解组内情况。

外企：需要英语水平较高，官网有时比较乱，可以到社交媒体（牛客、小红书）搜索。


+ 工作时间：

目前互联网行业没有真正的955，即便外企也要加班。


+ 岗位对比：

开发岗：做技术、基架会比业务部门压力小，但薪资和晋升速度也会略低。 

算法岗敲门砖：一篇一作高质量B会。

# 实战篇：面试中的“攻防转换”

##  2.1  面试流程与潜规则

面试其实是“命题作文”，流程基本固定

+ 自我介绍：3分钟版本，我是谁 + 我最牛的项目，没必要把简历上所有项目都详细说，可以详细说一个，剩下的简略一些。
+ 针对简历中项目、实习经历提问。
+ 八股、面试官提问。如果前面的项目和实习足够丰富，八股其实考的很少甚至没有。
+ 算法题。
+ 岗位介绍、闲聊、反问。

反问问题最好提前准备好，例如“如果我能顺利加入，希望我来承担什么工作？”
