# 准备篇：打好地基

## 1.1 简历构建与迭代

+ 简历的核心逻辑是：**在最短时间内，让 HR 看到你最匹配的价值。** 

+ 互联网简历侧重内容，信息清晰传达即可，无需过度美化，可适当加粗。

+ 推荐简历制作工具：[WonderCV](https://www.wondercv.com/)。

<br>

<v-click>

### 头部信息：清晰对齐
* **基本信息**：姓名、求职意向（开发 or 算法？后端 or 前端？）、联系方式、个人主页或github主页（加分项）。照片可放可不放。
* **教育背景**：学校、专业、绩点 (若有优势，也可写部分课程)、起止时间、所获荣誉（可选），按照时间倒序排序（即先写硕士再写本科）。
> **原则**：固定格式，无需设计感，确保信息易读。

</v-click>

### 核心板块

实习、项目、论文、专利、竞赛奖项。

+ 对于开发岗，实习和项目一般必须有，论文、专利、竞赛奖项根据实际情况；算法岗则更侧重论文。
+ 根据个人情况与要找的岗位、含金量，按照重要性排序。
+ 不需要罗列所有经历。只选最出彩的，每个板块只写2-3个项目并按重要程度。
+ 采用 **STAR 原则**，每条经历可以分成 3 个要点，侧重量化和结果。
+ 方法可以不用过于详细，面试时介绍详细解决方案。

```
编写 Go/Envoy 插件完成 Higress 网关 的 MCP 协议适配，将内部微服务以 MCP 暴露，支撑数百个智能体通过 MCP 协议直接调用内部微服务。

使用交叉编译重构插件构建链路并设计 CI 流程，插件单测耗时由约 10 分钟降至 1 分钟，解决由于 DinD 环境父子容器不一致导致 redis 丢包问题。
```

### 补充板块

个人技能、学生工作、社团经历、个人兴趣。

+ 个人技能可写3行左右，表明自己的技术栈，主要目标是**提高岗位匹配度**。
+ 研发岗对学生工作关注度较低。精简至 1-2 行，除非该经历能体现极强的组织能力或领导力。

```markdown
# 个人技能
容器技术与微服务架构：熟悉 Docker 和 k8s 容器技术及实现原理，曾使用 C++ 实现简易容器运行时；理解微服务架构
AI及Agent：理解 AI 模型和 Agent 基础，能够使用 pytorch 编写和调试神经网络代码，能使用 LangGraph 框架开发 Agent

# 组织经历
负责 NOIP、ICPC 等多个计算机竞赛的技术支持，赛前为选手安装并调试竞赛环境；竞赛中及时解答考生技术性问题，并及时处理赛场设备和网络故障。
```

##  简历总结&注意事项

个人基本信息+教育经历+核心板块（实习、项目、论文）+ 补充（社团、技能、学生工作）

<v-clicks>

- **拒绝"流水账"**：面试官不关心你"做了什么"（最多用一个点说明自己的工作内容），更关心你"解决了什么问题、起到了什么作用、达到了什么效果"。
- **量化意识**：将"优化了插件编译速度"改为"插件编译时间从10分钟降低到1分钟"。
- **技术栈对齐**：简历上的每一个字都可能成为考点，不熟悉的的项目、技能千万别随便写。
- **排版技巧**：一页纸原则，开头结尾一般固定，中间核心板块按照重要性排序。

</v-clicks>

<!--
总结一下，简历的组成部分其实是有一个固定格式的，只是每部分的顺序不同。组成部分就是个人头部（基本）信息+教育经历+核心板块+补充内容。

最后说一些注意事项，首先是简历拒绝流水账，还是刚才那句话，大家不要说一个经历去写了一大片的文字，而是要分成不同的点，每一点可能就一两行，每个经历两三个点这样去写。

第二点就是要有量化意识，不要说你提高了什么东西，减少了什么时间，而是说提高了百分之多少，将什么什么时间从几秒降到了几秒。

第三点就是说要注意自己不懂的东西，不要随便往上写，比如说你的技能部分或者说项目部分，如果说自己没有做过，比如说找的那种大家用的很多的项目，肯定也会去参考别人的简历啥的，但是不要看别人写了自己就写，如果自己没做过就不要往上写。

最后呢，就是计算机专业的简历还是简洁和信息化最重要，基本上是一页纸就足够，不要太花里胡哨。然后呢，中间的板块按照重要性进行排序，对于每一个板块的几个条目也是按照重要性来排序。

# 1.2 项目、实习、八股、刷题

## 项目

<v-clicks>

- 准备2~3个项目，不侧重数量，主要侧重质量。
- 可先根据自己个人兴趣做，如果没有合适的，可以从平台里面找。

</v-clicks>

<h2 v-click> 实习 </h2>

<v-clicks>

+ 目前互联网领域实习尤为重要，可在**完成论文第一个点征得张老师同意**后实习。
+ 建议尽快完成第一个点，实习经历需要3个月，而8月就会开始秋招面试。
+ 实习需要好好找，因为实习转正的难度远小于秋招，而且实习转正拿到的offer对团队知根知底，也更好一点。
+ 实习也可以多投一些，一方面积累面试经验，一方面实习offer有可能对于秋招很重要。

</v-clicks>

+ 需要多看看有个知识广度，但只有自己熟悉的东西才需要真的好好记和背，面试的时候往自己熟悉的领域引导面试官。

+ 推荐平台： [小林coding](https://www.xiaolincoding.com/)（官网）、黑马程序员（B站）。

## 算法刷题

+ LeetCode [hot100](https://leetcode.cn/problem-list/2cktkvj/)、[热题 100](https://leetcode.cn/studyplan/top-100-liked/)、[面试150](https://leetcode.cn/studyplan/top-interview-150/)，只要前面交流没啥问题，出题基本不会超出这个范围。
+ 如果题目难度明显很难，表明面试官意愿不太大，但如果真能做出来也会有转机。
+ 一定要刷熟练，如果只知道思路，面试时紧张不一定能调出来。
  
</v-click>

# 实战篇：面试中的"攻防转换"

##  2.1  面试流程与潜规则

面试其实是"命题作文"，流程基本固定

<v-click>

+ 自我介绍：3分钟版本，我是谁 + 我最牛的项目，没必要把简历上所有项目都详细说，可以详细说一个，可以提前准备。
+ 聊简历：针对简历中项目、实习经历提问。
+ 聊八股：八股、面试官提问。如果前面的项目和实习足够丰富，八股其实考的很少甚至没有。
+ 写题：算法题。
+ 结尾：岗位介绍、闲聊、反问。
  
</v-click>

<v-click>

反问问题最好提前准备好，例如：

+ "如果我能顺利加入，希望我来承担什么工作？"
+ "您对这个岗位的理想人选有什么期待？"
+ "团队目前的技术栈和未来的核心发展方向是什么？"

</v-click>

## 2.2 攻防转换：如何掌控面试节奏


<v-click>

+ “引导”的艺术：在回答 A 的时候，故意露出一个你很熟悉的 B 的引子。

例如： “我为项目编写了完整的CI/CD流程，并且解决了一个因为DinD环境MTU不一致导致网关无法收到Redis的BUG”（面试官大概率会接着问你BUG是什么，怎么解决）。

</v-click>

<v-click>

+ 遇到“不会”怎么办？

如果真的确实完全不了解，可以直接说不清楚。只要是稍微知道一些就可以说，例如： “这个底层实现我目前了解不深，但根据我的理解，它应该是基于 XX 原理实现的，类似于 YY...”

> 面试不是问答的考试，而是一场**技术交流**。

</v-click>

<!--
面试中你可以引导面试官向你熟悉的方向去问，当你在回答他的问题的时候呢，可以带一些自己熟悉的或者说自己的亮点,这样的话面试官看到之后会很自然的去询问，然后你就可以正好展开去说。

大家要注意面试和笔试不一样，面试不是一个考试，更像是一种面对面的交流，除了你能回答面试官的问题，你也可以主动的去说或者引导面试官问一些自己的亮点。

# 最终篇:挑选最合适的offer

## 3.1 Offer 抉择：建立“多维坐标系”

+ 团队氛围：如果是自己实习过的部分是最了解的，不是自己实习的部门可以社交媒体问问在那边工作的人。
+ 涨薪幅度、晋升速度 vs 生活节奏、工作压力。
+ 总包、月base，后续涨薪是按照月base来，同样是年薪60w，但是`30k*20 < 38k*16` 。
